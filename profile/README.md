# Hiperbaric

Engineering technology for high pressure applications.

Hiperbaric develops industrial solutions around high pressure processing, hydrogen compression and advanced pressure technologies. This GitHub organization hosts internal and shared software projects that support engineering, operations, automation, data, AI and digital workflows across the company.

## What We Build

- Internal tools for productivity, automation and knowledge access.
- Integrations with business systems, Microsoft 365, Azure and operational platforms.
- AI assistants and retrieval systems for company documentation and support workflows.
- Data processing utilities for logistics, purchasing, service and engineering teams.
- Developer tooling and reusable components for Hiperbaric projects.

## How We Work

We value practical software that is reliable, maintainable and useful for real teams. Projects in this organization should be clear about their scope, configuration, deployment process and operational ownership.

When contributing, keep changes focused, document important decisions and avoid committing secrets, credentials or customer data. Use environment variables, Key Vault or the approved secret-management process for sensitive configuration.

## Repository Guidelines

Each repository should include:

- A clear `README.md` with purpose, setup and run instructions.
- An `.env.example` or equivalent configuration reference without secrets.
- Tests or validation steps for the main workflow.
- Deployment notes when the project runs in Azure, Microsoft 365, GitHub Actions or another managed environment.
- Ownership or contact information for operational support.

## Useful Resources

- Website: [hiperbaric.com](https://www.hiperbaric.com/)
- Internal documentation: use the approved Hiperbaric knowledge and document repositories.
- Issues and pull requests: use the repository-specific workflow unless a project states otherwise.

## Security

This organization may contain private repositories and internal tooling. Do not publish confidential business information, personal data, credentials, access tokens, customer files or proprietary documents.

If you find a security issue, report it through the internal security or IT channel instead of opening a public issue.
